计算机系统概要

计算机组成

首先，可以把计算机大致分为：CPU、内存、外部设备三种结构。外部设备简称为外设，不单单包括键盘、鼠标、显示器这些平时接触比较多的设备，还包括外部存储器、网络适配器这些设备。
在计算机体系结构中，越靠近CPU的设备，要求的速度和性能就越高，当然造价也越昂贵，这就导致内存、缓存这些靠近CPU的设备的存储规模不能过大（一方面过大会导致计算机的造价昂贵，另一方面会导致速度和性能的下降）否则就会拖累CPU的性能。

but why?

因为计算机在处理数据的过程中是一个：

CPU从内存中取指令、译码、寻址、执行

把执行的结果重新写入内存

内存中的数据写入外存

通过网络适配器发给其他主机或者通过外部设备显示

循环往复的过程。像上楼梯一样，前面的人不走，后面的人就走不了。计算机的设备也是一样的道理，和CPU交互的设备响应速度如果很慢的话，CPU就要耗费性能去等待，这是不可以容忍的。因为CPU是不会停止工作的，不是说等待就可以把电源关闭停止耗能了，CPU的等待其实是在不断执行空操作nop。我们要充分利用CPU的性能，尽可能地减少这种等待。

操作系统

上面介绍的设备都隶属于硬件，而操作系统（Operating System, OS）属于是计算机的系统软件：可以直接控制硬件设备，同时为应用程序提供运行环境并给用户提供交互（不单是指GUI交互，有的OS没有GUI交互界面，只有CLI命令行交互环境）。

进程

进程是计算机资源分配的最小单位。一般一个程序中会有多个进程，并以进程为单位在OS上运行。

设备驱动程序

进程在执行的过程中，难免地需要调用某些外部设备，如果不加以管制和处理，不仅开发成本高、而且可能会遇到多个进程访问同一个设备的情况，这样势必会导致对设备资源的竞争，出现难以预估的后果（多次运行同一程序但是结果不一致，破坏了程序要求的唯一性；甚至会导致读、写顺序不一致引起计算结果的错误！）。

为了解决上述问题，OS把设备的调用整合成一个设备驱动程序，使得进程通过设备驱动程序来调用设备。设备驱动程序分成好多类（输入输出、外部存储、网络适配等），同一类型的设备通过同一个设备驱动程序的接口来调用。

内核模式和用户模式

但是，如果某些程序员恶意地使进程直接调用设备，而不经过设备驱动程序，那岂不是依然会出现多个进程调用同一设备的情况？又该如何处理呢？

OS借助硬件，将CPU分为内核模式和用户模式两种工作模式。进程工作在用户模式下，设备驱动程序工作在内核模式下，并且只有处于内核模式下才允许访问设备。如此一来，工作在用户模式下的进程只得通过调用工作在内核模式下的设备驱动程序来访问设备。

内核

不单单有设备驱动程序不应该直接由进程调用而工作在内核模式下，进程管理系统、进程调度器、内存管理系统等也都工作在内核模式下。这些在内核模式下运行的OS的核心就是由内核整合在一起的。普通进程如果需要访问设备，就需要向内核发出请求，这种请求就称作系统调用。

回到这一小节的标题：操作系统——不单单指的是内核，而是内核和许多工作在用户模式下的程序所构成。在OS中，内核管理着CPU和内存等资源，按需分配给系统上运行着的各个进程。CPU管理由进程调度器负责；内存管理由内存管理系统负责（后续章节将会陆续学习）。

文件系统

如果需要访问外存中的数据，理论上讲可以通过设备驱动器来访问，但是过程需要调用外部存储器驱动程序提供的系统调用，然后再由外部存储器驱动程序访问外部存储器，甚是复杂！实际上OS中引入了文件系统程序来访问外部存储器中的数据。

为什么外部存储器对于计算机来讲不可或缺？

事实上，计算机在开关机的过程中经历了许多操作：在启动的过程中，首先由基本输入输出系统（BIOS，Basic Input Output System）初始化硬件设备，引导存储在外部存储器中的OS；在关闭电源时需要把内存中的数据写入外部存储器中以防止掉电丢失（内存的物理性质属于掉电易失性）。

用户模式实现的功能

前面提到：普通进程是工作在用户模式下的，如果需要访问内核，需要通过系统调用来发起请求。而系统调用方式并非一种，而是由：进程独有的代码直接发起系统调用（系统调用）、通过OS提供的库发起调用（OS提供的库）、通过第三方库发起调用（如C标准库等）。

系统调用

系统调用包括：进程控制（创建和删除）、内存管理（分配和释放）、进程间通信、网络管理、文件系统操作、文件操作（访问设备）。进程需要依赖于内核进行处理，都需要通过以上系统调用来发起请求。

CPU的模式切换

通常程序运行在用户模式，当通过系统调用向内核发起请求时，CPU会发生中断，并由用户模式切换到内核模式，然后根据请求的内容进行响应，当事件处理完毕之后再重新切换回用户模式继续运行原来的程序。具体流程如下图所示：

系统调用的包装函数

系统调用不能由C语言之类的高级语言直接发起，只能通过和系统架构紧密相连的汇编语言代码发起。如果没有OS，那么每个程序员都必须清楚地了解计算机的体系架构和汇编语言代码，开发效率极低，同时也无法移植。因此，OS提供了一些列系统调用的包装函数的函数，用于在系统内部发起系统调用，不同架构存在对应的包装函数。程序只需要调用OS提供的包装函数即可。

C 标准库

C 语言拥有 ISO（International Standard Organization，国际标准化组织）定义的标准库，Linux 也提供了这些 C 语言标准库。

OS提供的程序

OS 提供的程序与 OS 提供的库一样，对绝大多数程序来说是不可或缺的。OS 同时还提供了作为自身一部分的、用于更改 OS 运行方式的程序。
初始化系统：init
变更系统的运行方式：nice、sync、sysctl
文件操作：touch、mkdir
文本数据处理：grep、sort、uniq
性能处理：sar、iostat
编译：gcc

进程管理

本章节抛开第五章的虚拟内存来讲进程的创建和删除。

创建进程

创建进程的目的：将同一程序分成多个进程执行；创建另一个程序。Linux提供了fork()与execve()函数来实现。

fork()函数

调用fork()函数后，会基于发起调用的进程，创建一个新的进程，发出请求的是父进程，新创建的进程是子进程:
① 为子进程申请内存空间，并复制父进程的内存到子进程的内存空间。
② 父进程与子进程分裂成两个进程，以执行不同的代码。这一点的实现依赖于 fork() 函数分别返回不同的值给父进程与子进程。
